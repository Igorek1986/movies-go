# movies-api

Монолитный Go-бэкенд для экосистемы **Lampa** / **Lampac**. Объединяет в одном сервисе:

- **Парсер контента** — `rutor.info` / `kinozal.tv` / `nnmclub.to` → обогащение метаданными через **TMDB** → выдача в формате **Lampac API** (источник для плагинов Lampa).
- **Управление пользователями** — авторизация, устройства, таймкоды, импорт таймкодов из **MyShows**, **Telegram-бот**.

Парсер основан на проекте [YouROK/NUMParser](https://github.com/YouROK/NUMParser).

## Возможности

**Контент и каталог**
- Многотрекерный парсер (rutor / kinozal / nnmclub) с инкрементальным обходом и устойчивостью к падению трекеров
- Обогащение через TMDB: постеры, рейтинги, жанры, актёры и режиссёры, эпизоды сериалов
- Богатый каталог в формате Lampac: жанры, новинки, 4K, русское, аниме, мультфильмы, подборки по актёрам/режиссёрам/годам, «продолжить просмотр»
- Полнотекстовый поиск по каталогу, скрытие просмотренного

**Пользователи и устройства** (режим `all`)
- Аккаунты с 2FA, управление сессиями, восстановление пароля через Telegram
- Активация устройств Lampa по коду, лимиты по ролям (simple / premium / super)
- Таймкоды и история просмотра, синхронизируемые с сервером («продолжить просмотр»)
- Импорт отметок просмотра и таймкодов из MyShows
- Экспорт/импорт личных данных (резервная копия в JSON)

**Интеграции и управление**
- Telegram-бот: привязка аккаунта, уведомления (вход, истечение премиума), поддержка
- Веб-панель администратора: статистика, настройки, парсеры, карточки, прокси, модерация

## Стек

- **Backend:** Go, chi router, PostgreSQL (без ORM, чистый pgx)
- **Frontend:** React + Vite, собирается и **встраивается** в бинарь
- **Плагины:** отдаются из `./plugins/` в рантайме (hot-reload, без пересборки)

## Плагины

JS-плагины для клиента Lampa отдаются из `./plugins/` (URL задаётся в админке, параметр `plugin_url`). Готовые плагины, включая источник контента `np.js`, поддерживаются отдельно: <https://igorek1986.github.io/lampa-plugins/>.

## Режимы работы

| Режим | Что включено |
|-------|--------------|
| `parser` (по умолчанию) | только контент: парсер + Lampa API + админка парсера |
| `all` | всё: парсер + веб-кабинет + аккаунты + Lampa API |

Режим хранится в БД (`app_settings.app_mode`), переключается в веб-панели и **требует перезапуска**.

## Быстрый старт

Нужны **Docker** и **Docker Compose**.

```bash
git clone git@github.com:Igorek1986/movies-go.git
cd movies-go
cp .env.example .env
nano .env                       # укажи TMDB_TOKEN и SUPERUSER_*
docker compose up -d --build
```

Сервис поднимется на `http://localhost:8888` (вместе с PostgreSQL).

> Либо одной командой через установочный скрипт: `./scripts/install.sh` — проверит зависимости, создаст `.env`, по желанию зальёт дамп карточек и поднимет сервис.

## Бутстрап базы из дампа

Чтобы не парсить каталог с нуля, можно залить готовый дамп карточек из последнего релиза. Скачиваем `cards-dump.sql.gz` и запускаем скрипт восстановления (он сам найдёт файл, применит схему, восстановит данные и соберёт приложение):

```bash
cp .env.example .env            # отредактируй перед запуском
curl -L -O \
  https://github.com/Igorek1986/movies-go/releases/latest/download/cards-dump.sql.gz
./scripts/restore.sh
```

`cards-dump.sql.gz` — **публичный** дамп: только контент карточек (`media_cards`, `torrents`, `episodes`) и метки последнего парсинга. Без пользователей, токенов и прочих секретов.

> [!NOTE]
> Метки последнего парсинга (`*_last_parsed_at`) включены в дамп, поэтому после восстановления парсер продолжит обход с момента создания дампа, а не пойдёт сканировать каталог заново. Уже имеющиеся карточки обновляются идемпотентно, без дублей.

## Перенос на другой сервер

Дамп карточек переносит только каталог. Чтобы перенести сервис **целиком** (пользователи, устройства, таймкоды, настройки), сделай полный бэкап всей базы.

```bash
# на старом сервере — полный дамп всей БД (включая пользователей и секреты)
./scripts/backup.sh full-backup.sql.gz

# перенеси на новый сервер также .env и каталог plugins/
scp full-backup.sql.gz .env user@new-server:/path/to/movies-go/
scp -r plugins user@new-server:/path/to/movies-go/
```

На новом сервере:

```bash
git clone git@github.com:Igorek1986/movies-go.git
cd movies-go
# положи сюда .env, plugins/ и full-backup.sql.gz
docker compose up -d db
gunzip -c full-backup.sql.gz | docker exec -i movies-api-db psql -U movies_api -d movies_api
docker compose up -d --build app
```

> [!WARNING]
> `full-backup.sql.gz` содержит **всё**, включая пароли, токены и данные пользователей. Храни приватно и никогда не публикуй — для открытого шаринга каталога есть только `cards-dump.sql.gz`.

## Админка и настройки

Веб-панель доступна в **обоих режимах**, но объём функций разный. В режиме `parser` вход — по отдельной парсер-сессии (суперпользователь из `.env`), система аккаунтов не используется. В режиме `all` — вход под админ-аккаунтом.

<details>
<summary><b>Доступно в обоих режимах (parser + all)</b></summary>

- **Настройки** — общие параметры сервиса, тексты, лимиты, интеграции (включая токен Telegram-бота и доступ к kinozal — задаются здесь, **не** через env)
- **Парсеры** — запуск/остановка общего цикла, запуск и сброс по каждому трекеру, порядок и параметры ретраев
- **Карточки** — список за сегодня и полный каталог (серверная пагинация и фильтры), удаление, правка дат
- **TMDB missing** — карточки, которые не удалось сопоставить с TMDB, с возможностью удаления
- **Модерация** — бан-паттерны и ключевые слова детского контента (фильтрация по возрасту)
- **Системная статистика** — нагрузка, состояние сервиса
- **Прокси** — список прокси и правила маршрутизации запросов

</details>

<details>
<summary><b>Добавляется в режиме <code>all</code> (полный сервис)</b></summary>

- **Аккаунты** — регистрация, авторизация, сессии, 2FA, сброс пароля
- **Устройства** — привязка устройств Lampa, веб-кабинет таймкодов и истории
- **Импорт из MyShows** — перенос отметок просмотра и таймкодов
- **Telegram** — привязка аккаунтов, уведомления, управление ботом
- **Пользователи** — роли, премиум-лимиты, блокировки, массовое продление премиума
- **Статистика** — пользователи, устройства, запросы по категориям и эндпоинтам, синхронизации MyShows
- **Контент-инструменты** — фоновое обновление карточек, дозагрузка актёров, фикс длительности, обновление эпизодов
- **Логи** — просмотр и стрим логов сервиса

</details>

## Конфигурация

Через `.env` задаётся только базовый минимум (см. [`.env.example`](.env.example)). Всё остальное — в веб-панели.

| Переменная | Описание |
|------------|----------|
| `TMDB_TOKEN` | Bearer-токен TMDB (без него обогащение метаданными выключено) |
| `SUPERUSER_USERNAME` / `SUPERUSER_PASSWORD` | админ/суперпользователь, создаётся при первом старте |
| `PORT` | порт HTTP (по умолчанию `8888`) |
| `DB_USER` / `DB_PASSWORD` / `DB_NAME` | параметры PostgreSQL (по умолчанию `movies_api`) |

> Токен Telegram-бота, доступ к kinozal, параметры трекеров, лимиты и тексты указываются **в настройках админки** (раздел «Настройки»), а не в `.env`.
